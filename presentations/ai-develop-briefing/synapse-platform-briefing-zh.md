## 执行摘要（顶层答案）

### Synapse Platform 是什么

Synapse Platform 是一套 **AI 驱动的数字分身协作系统**，构建于企业已普遍使用的 Slack 即时通讯平台之上。系统的核心逻辑是：**将老板/PM 的工作意图，通过 AI 自动转化为可审批的高质量交付物**，同时保留人工对每一项输出的最终裁量权。

用一句话概括：**Synapse 是一位随时在线、不需要培训、越用越聪明的 AI 协作助理，而老板始终握着审批权。**

### 三大核心价值

1. **降低高频工作负担**
   周报撰写、会议材料准备、研究摘要——这些重复性、耗时的工作，由 AI 在 Slack 内即时完成，无需人工从零开始。

2. **知识沉淀与持续进化**
   每一次人工审批，都自动成为系统训练数据。系统具备 4 层个人训练机制，越用越贴合实际工作风格，无需额外维护成本。

3. **审批权完整保留，全链路可追溯**
   AI 产出必须经人工审批后才能落地，所有操作均记录在不可变审计日志中。系统提供完整的决策证据链，满足合规要求。

## 一、业务能力全景（归纳支撑层）

Synapse Platform 的 20 项核心能力，按业务价值归为四个层次。

### 第一层：接入与理解（3 项）

**能力 1：无缝接入 Slack，零改造成本**
系统通过 WebSocket 长连接接入 Slack，不需要开放任何公网端口，不影响企业现有 IT 安全策略。员工和老板通过日常使用的 Slack DM（私信）发出请求，无需学习任何新工具。

**能力 2：智能双层意图理解**
系统能够自动将用户请求分类为 7 种业务场景（周报、会议材料、研究分析等）。分类采用「关键词即时匹配 + AI 语义兜底」双层机制：常规请求毫秒内完成分类；模糊请求触发 AI 语义理解，确保分类准确率。

**能力 3：文档驱动知识自动入库**
企业知识不再停留在 Google Drive 里无人问津。系统自动监控指定的 Google Drive 文件夹，识别 PDF、Word、Excel、PPT、Markdown 等格式，提取关键信息，通过语义向量技术建立知识库，并同步回写至团队 Obsidian 知识库，实现「文档即知识」的自动化闭环。

### 第二层：执行与协作（7 项）

**能力 4：周报智能生成**
PM 只需在 Slack 发送「本周周报」，系统调用 PM Agent，基于历史工作记录和个人工作风格，生成符合金字塔结构的中文周报草稿，人工确认后一键发出。

**能力 5：会议材料快速准备**
会前30分钟不再手忙脚乱。PM Agent 基于会议主题和背景文档，按照 SCQA（情境-冲突-问题-答案）框架，自动生成结构清晰、论点突出的中文会议材料。

**能力 6：研究请求与数据分析**
遇到需要快速市场调研、竞品分析的场景，Research Agent 采用「MECE 分类 + 假设驱动」方法论，输出英文研究报告，有理有据，避免信息遗漏。

**能力 7：内容草稿与决策简报**
Content Agent 负责对外内容撰写和内部决策简报，采用亚马逊 6-pager 格式，英文输出，适合国际化工作场景。

**能力 8：内部服务请求处理**
系统日常运维、内部工具配置、服务请求等事务，通过 Service Agent 统一处理，减少跨团队沟通成本。

**能力 9：多 Agent 协作——中英文无缝切换**
这是系统最具差异化的能力之一。当老板用英文发出指令，BossAgent 将任务分解后交给 PMAgent 用中文起草，再翻译为英文交付老板。**双语协作摩擦归零，老板和 PM 各说各的语言，系统负责中间的翻译和协调。**

**能力 10：Slack 内联编辑与即时训练**
人工审批不是简单的「同意/拒绝」。系统在 Slack 内提供「编辑并批准」弹窗，审批人可直接修改 AI 草稿。**每一次修改，自动成为系统的个人训练样本**，系统从中学习「这位老板/PM 更喜欢怎样的表达方式」，形成持续进化的个人风格记忆。

### 第三层：治理与质量（5 项）

**能力 11：确定性工作流引擎（FSM）**
系统采用自研有限状态机（FSM），将每个任务的生命周期分为 8 个明确状态（从「接收请求」到「完成交付」），每一步均强制记录审计信息。**工作流逻辑确定、可预测、可追溯，不存在 AI 「自作主张」跳步的风险。**

**能力 12：强制人工审批门禁**
任何 AI 输出，未经人工审批，系统技术层面禁止其进入「已完成」状态。这不是流程规范，是技术硬约束，确保审批权始终在人手中。

**能力 13：SLA 监控与主动告警**
系统对每个任务设置服务级别目标（SLA）。临近截止时间自动推送 Slack 预警；一旦超时，立即发出违约告警，并按任务优先级动态调整告警频率。老板对团队工作状态保持实时可见性，无需主动查询。

**能力 14：不可变审计日志**
系统记录 30 余种事件类型，形成完整的「证据包（EvidenceBundle）」，任何历史操作均不可篡改。满足合规场景下的操作留痕要求。

**能力 15：持续集成/持续交付（CI/CD）**
每次代码变更，自动触发完整测试流程。当前测试通过率 294/295，涵盖单元测试、集成测试、验收测试、冒烟测试、端到端测试，确保每一次迭代的系统稳定性。

### 第四层：智能进化（5 项）

**能力 16：语义知识检索（RAG）**
系统采用 Voyage AI 语义向量模型，配合向量数据库（pgvector）和 Jina AI 重排算法，基于 Voyage AI 语义向量 + pgvector 向量库 + Jina AI 重排，**专项优化中文理解**，确保中文业务语境下的知识检索准确率。

**能力 17：四层个人训练系统**
- **Layer 1 语料层**：收录所有历史交付物
- **Layer 2 黄金样本层**：人工批准且高质量的样本，作为风格标杆
- **Layer 3 规则层**：人工明确定义的偏好规则（如「避免过于正式的开场白」）
- **Layer 4 渐进记忆层**：系统自动提炼的偏好摘要（上限 50 条）

四层协同，形成越用越贴合个人风格的 AI 助手。

**能力 18：Web 管理看板**
系统提供独立的 Web 管理界面（端口 3001），支持案例管理、知识文档上传、偏好规则配置（CRUD）、Google Drive 连接配置等，无需技术背景即可操作。

**能力 19：Slack 查询命令**
通过 `/synapse cases`、`/synapse case [ID]`、`/synapse stats` 等命令，在 Slack 内即时查询案例列表、案例详情、系统使用统计，无需切换其他工具。

**能力 20：权限系统预埋（RBAC）**
系统已完成角色权限控制（RBAC）的技术预埋，支持 Phase 5 正式启用多角色、多 Workspace 场景，为企业规模化部署奠定基础。

## 二、架构与技术实力（演绎支撑层）

### 核心架构决策：自研引擎，确定性优先

在系统架构层面，团队做出了一个关键决策（ADR-0001）：**明确拒绝使用市场主流的 LangGraph 框架，选择自研工作流引擎。**

理由如下：
- LangGraph 的图计算模型适合研究场景，但在生产环境中存在不确定性——AI 可能选择不同的执行路径，导致行为难以预测和测试。
- 自研有限状态机（FSM）保证了每个任务的执行路径完全确定，每一步均可单独测试和验证，每条路径单独可测，确定性收敛。
- 决策结论：**牺牲灵活性，换取可靠性和可追溯性。这是 B2B 场景的正确取舍。**

### 核心技术选型

| 组件 | 技术选型 | 选型理由 |
|------|----------|----------|
| 消息接入层 | Slack Bolt + Socket Mode | 无需公网端口，企业安全友好 |
| AI 推理层 | Anthropic Claude claude-sonnet-4-6 | 最强综合推理能力 |
| 向量存储 | PostgreSQL + pgvector | 生产级稳定性，无额外基础设施 |
| 语义编码 | Voyage AI voyage-3 | 中文语义优化，精度领先 |
| 重排算法 | Jina AI Reranker | 检索结果精度提升 |
| 文档接入 | Google Drive API | 零改造，复用现有文档体系 |
| 管理界面 | Express.js + Web Dashboard | 轻量、快速迭代 |

### 架构弹性设计：开发与生产一键切换

系统采用适配器模式（Adapter Pattern）设计存储层：
- **开发/测试环境**：使用本地文件存储（File Store），零依赖，启动即用
- **生产环境**：切换为 PostgreSQL，通过单一环境变量 `STORAGE_BACKEND` 控制

由此实现了「本地快速开发 → 生产稳定运行」的平滑过渡，无需修改任何业务逻辑代码。

### AI 成本控制

系统对 Claude API 调用实施 Prompt Cache 机制，将系统提示词、知识上下文等高频不变内容缓存，有效控制缓存块数量在 4 个以内，**在保证推理质量的同时，显著降低单次 API 调用的 Token 成本。**

## 四、战略价值与 Roadmap

### PDCA 闭环：AI 系统的持续进化机制

Synapse Platform 的核心价值不仅在于当下能做什么，更在于其内置的持续进化机制：

```
Plan（规划）：老板/PM 在 Slack 发出业务请求
    ↓
Do（执行）：AI Agent 自动生成草稿，附完整推理依据
    ↓
Check（检查）：人工审批，可直接在 Slack 内编辑修改
    ↓
Act（改进）：审批记录自动进入训练系统，系统持续进化
    ↓
回到 Plan…（下一次请求，AI 表现更贴合实际需求）
```

**每一次日常使用，都是一次系统进化的机会。** 这与传统 AI 工具「一次部署、永久不变」的模式有根本区别。

## Page 1 — 核心架构：让业务专家用业务语言驱动 AI

### 问题：AI 工具强大，但"会用"的门槛太高

大多数 AI 工具的使用路径是：业务需求 → 技术翻译 → 提示词工程 → AI 执行。
每一层翻译都损失信息，每一次执行都需要人工盯盘，每一个新场景都要重新配置。

**结果是：AI 在帮助工程师，却没有在帮助真正驱动业务的人。**

### 解决方案：Synapse — 行为可审计、能力可积累、结果可复制

Synapse 的设计哲学是：**业务专家用业务语言提出目标，AI 完成从分析到交付的全部过程，结果经过质量门禁，过程留下可审计记录。**

Owner 参与度：**100% → 0%**（仅接收 Slack 结果通知，不参与执行过程）

### 核心公式

```
Agent = Model + Harness

Model   = 大语言模型（Claude claude-sonnet-4-6）
Harness = 规则 + 约束 + 流程 + 记忆 + 角色定义

Harness 决定 Agent 的专业度、可靠性与可预测性。
```

### 典型使用场景

**场景 A：新产品全周期自主开发（Phoenix 模式）**

> Owner 提出目标 → Lysander CEO 承接 → 团队自主执行 4 天 → 3 个关键节点 Owner 确认 → GA 交付

关键数字：**4 天连续执行** · Owner 参与 **3 个节点** · 测试通过率 **294/295（99.7%）**

**场景 B：日常运营自动化（情报管线模式）**

> 每日 6am 任务恢复 → 8am 情报日报自动生成 → 10am 情报行动执行 → Slack 通知 Owner 摘要

Owner 只需阅读每日 Slack 摘要；L4 项总裁知会，L5 项总裁裁决。

**场景 C：知识积累与决策复用（OBS 记忆模式）**

> 每次决策 → D-编号归档 → 新会话自动读取历史上下文 → 避免重复犯错 → 知识持续沉淀

**35 条**跨会话持久记忆，确保系统每次启动都拥有完整历史背景，不从零开始。

# D-2026-0512-001：Worktree → Main 强制同步规则（P0 防护机制）

## Decision

Lysander CEO + 智囊团（L4，知会总裁）批准建立 Worktree → Main 强制同步规则，并上线配套 PowerShell 脚本，防止 Claude Code worktree 分支中的知识工作对 Obsidian 永久不可见。

## Context

发现 Claude Code 每次会话自动创建 `claude/<name>` worktree 分支，所有写入 `obs/` 的文件仅存在于该分支，不自动合并到 main。Obsidian 读取 main 分支，导致数月的知识工作对总裁完全不可见。发现时约有 ~575 commits 分布在 5 个分支中从未合并。

## Options Considered

**选项 A：手动合并（现状）**
依赖记忆，已证明会积累数月不处理。P0 风险。

**选项 B（本决策）：Harness P0 规则 + 同步脚本**
规则层面强制，脚本自动化执行，支持 DryRun 预览模式。

## Decision Result

选择选项 B。执行内容：
- `scripts/sync-worktrees-to-main.ps1`：自动合并所有 claude/* 分支到 main，支持 -DryRun
- `.claude/harness/worktree-sync-rule.md`：P0 规则，含触发条件和 Session-End Checklist
- `CLAUDE.md`：changelog 增加本条 L4 记录

## Follow-Up

weekly-audit 检验同步规则执行情况；连续 2 周无执行记录 → L4 上报。
